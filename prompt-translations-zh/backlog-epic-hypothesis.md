# 产品待办 Epic 假设模板

<!--
## 描述：
此模板旨在促进产品待办事项 Epic 的创建，部分受 Tim Herbig 假设框架的启发。它提供了一种结构化方法来阐述关于新功能、产品或服务的假设。通过清晰地概述针对目标受益者的行动或解决方案、预期结果以及测试和验证的方法，团队可以更好地应对产品开发中固有的不确定性。

## 使用说明：
在生成式 AI 会话中使用此模板之前，请确保会话已预加载以下背景信息：
    - 提议的具体行动或解决方案。
    - 此行动或解决方案的目标受益者。
    - 受益者的预期理想结果或待完成任务。
    - 测试假设的潜在实验想法。
    - 验证假设的标准，包括定量和定性衡量标准。

## 使用说明：
1. 确认会话包含上述必要的背景信息。
2. 使用以下提示词来促进创建 Epic 假设：

"根据提供的背景，请使用以下 Epic 假设模板，以 Tim Herbig 假设的风格为产品待办事项创建一个 Epic，以 Markdown 代码块的形式呈现。"

```markdown
## 产品待办 Epic 模板

<!-- 使用此模板为你的产品待办事项构建 Epic，遵循 Tim Herbig 的精益用户体验假设方法 -->

### If/Then 假设

**如果我们** [代表目标用户角色的行动或解决方案]
**为** [行动或解决方案的目标用户角色]
**那么我们将** [为目标用户角色实现或达成理想的结果或待完成任务]。

### 微小的发现实验

**我们将通过以下方式测试我们的假设：**
* [实验 1]
* [实验 2]
* [根据需要添加更多实验]

### 验证衡量标准

**我们知道我们的假设是有效的，如果在** [以天或周为单位的时间框架]
**我们观察到：**
* [理想的可量化、可衡量的结果]
* [理想的可衡量的定性结果]
* [根据需要添加更多结果]
```
