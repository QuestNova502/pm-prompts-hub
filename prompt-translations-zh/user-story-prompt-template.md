# 用户故事提示词模板

## 描述：
本模板受 Mike Cohn 的用户故事用例格式和 Gherkin 验收标准格式启发，旨在帮助产品经理和开发团队创建清晰、简洁的用户故事。它指导用户故事的制定，确保其结构化以促进理解和可操作的结果。此模板特别适用于生成式 AI 会话，假设已经提供了足够的产品细节、画像洞察、定位和问题背景。

## 使用说明：
最适合在已建立全面背景信息（产品细节、画像、定位和问题）的生成式 AI 会话中使用。这确保生成的用户故事基于项目的背景和目标。

在生成式 AI 会话中使用此模板之前，请确保您的会话已具备以下背景：

1. **产品概览**：产品或功能的详细描述，用于创建用户故事。
2. **目标用户画像**：关于用户是谁的洞察，包括他们的目标、需求和痛点。
3. **产品目标**：产品或功能旨在实现的目标。
4. **竞争格局**：对市场中类似产品的理解以及产品如何脱颖而出。

确保上述背景提供后，请按照以下步骤操作：

1. 复制下面的用户故事格式/模板部分。
2. 输入提示词："基于提供的背景，请使用以下用户故事模板创建标题为'功能 XYZ'的用户故事，以代码块中的 Markdown 格式呈现。"
3. 将用户故事格式/模板粘贴到会话中，用与'功能 XYZ'相关的具体细节替换占位符文本。

这种方法确保生成的用户故事内容丰富、重点突出，并与产品的目标和用户需求保持一致。

## 用户故事格式/模板

```markdown
# 用户故事：[功能名称]

## 故事标题
[简短、描述性的标题]

## 作为（用户角色）
作为 [具体用户角色/画像]

## 我想要（目标）
我想要 [具体的功能或能力]

## 以便（价值）
以便 [实现的业务价值或用户收益]

## 验收标准

### 场景 1：[场景名称]
**假设（Given）** [初始状态或前提条件]
**当（When）** [用户执行的操作]
**那么（Then）** [预期的结果]

### 场景 2：[场景名称]
**假设（Given）** [初始状态或前提条件]
**当（When）** [用户执行的操作]
**那么（Then）** [预期的结果]

### 场景 3：[场景名称]
**假设（Given）** [初始状态或前提条件]
**当（When）** [用户执行的操作]
**那么（Then）** [预期的结果]

## 附加说明
* [任何相关的背景信息、限制或依赖关系]
* [与其他用户故事或系统组件的链接]

## 定义完成（Definition of Done）
- [ ] 代码已审查
- [ ] 单元测试已通过
- [ ] 集成测试已通过
- [ ] 文档已更新
- [ ] 产品负责人已接受

## 优先级
[高/中/低]

## 估算
[故事点数或其他估算指标]
```
